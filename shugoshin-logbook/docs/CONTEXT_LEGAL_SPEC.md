# Legal Spec (取適法 2026)
- 待機時間の記録を不変（Immutable）に保つ。
- 到着・出発の証拠（エビデンス）を自動生成。
# 取適法 (中小受託取引適正化法) 準拠要件

## 目的
運送業者が荷主に対して待機料を確実に請求できるよう、法的に有効なエビデンス（証拠）を自動生成する。

## 絶対要件 (Fail-safe)
1. **タイムスタンプの信頼性**: クライアント（ブラウザ/スマホ）の時刻は改ざん可能なため、打刻は必ずSupabase側の `now()` またはRPCのサーバー時刻を使用する。
2. **位置情報によるロック (Geofencing)**: 現場（指定された緯度経度）から500m以上離れている場合は、UI上で打刻ボタンを非活性（disabled）にし、エラーメッセージを表示する。
3. **トランザクションの確実性**: 到着打刻（発行）から作業完了打刻（クローズ）までの一連の流れを状態遷移（status）として厳格に管理する。
